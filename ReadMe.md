Архитектура Проекта
com.petprojects.projectbanking
│
├─ controller      # REST контроллеры (Auth, User, Admin, Transaction)
├─ service         # бизнес-логика
├─ repository      # JPA репозитории
├─ model           # сущности (User, Role, Account, Transaction)
├─ security        # конфигурация Spring Security, JWT или Session, DateCondit
├─ dto             # DTO для запросов/ответов
└─ exception       # кастомные исключения


Для Администратора(ADMIN):
Примечание. Администратор это один Аккаунт для одного главного
человека владельца, руководителя или топ-менеджера Банка. Создаётся этот Аккунт
в самом коде проекта, а не в реализации самого сайта, как это
например деалет при создании менеджера и пользователя. Там же,
в коде, он выбирает для себя Имя, Фамилию, и Номер пользователя
по которому он будет заходить в систему.
Администратору доступно:
1.Создание Менеджеров
2.Просмотр списка всех пользователей(Admin, Support, User)
4.Возможность отправлять письма с указаниями менеджерам
5.Просмотр данных своего Акакунта

Для Менеджера(SUPPORT):
1.Создание аккаунтов для пользователей
2.Приём запросов от пользователей на взятие кредита
3.Приём запросов о блокировке аккаунта
4.Приём запросов о разблокировке аккаунта
5.Приём запросов на удаление аккаунта
6.Просмотр аккаунтов пользователей, а именно:
По номеру пользователя, менеджер получает доступ к информации
- Имя, Фамилия, Баланс, Транзакции(Кто и кому прислал и отправил сколько денег и точное время и дата транзакций)
- Точное время и дата создания аккаунта
  7.Просмотр данных Акакунта(Статус(Менеджер), Имя, Фамилия, Номер счёта)

Для Пользователя(USER):
1.Просмотр баланса
2.Просмотр данных аккаунта(Статус(Пользователь), Имя, Фамилия, Баланс, Номер счёта)
3.Просмотр история транзакций(без времени и даты, только от кого пришли деньги)
4.Переводы денег/получение денег
5.Просмотр уведомлений/почта
6.Меню пользователя:
- Запрос на кредит
- Запрос на блокировку счёта
- Запрос на разблокировку счёта
- Запрос на удаление аккаунта
